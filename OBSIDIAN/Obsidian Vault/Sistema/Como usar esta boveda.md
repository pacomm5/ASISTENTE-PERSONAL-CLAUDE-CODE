# Cómo usar esta bóveda

Esta bóveda funciona como un segundo cerebro: un sistema externo que captura, organiza y conecta el conocimiento para que no tengas que retenerlo todo en la cabeza.

---

## El ciclo completo

### 1. Capturar

Cuando aparece algo (idea, información, tarea, reflexión), lo primero es capturarlo sin filtro. No importa el formato ni la estructura.

- Si estás en Obsidian: ve a [[Inbox/Captura rapida|Captura rápida]] y añade una línea.
- Si estás fuera: usa el método que menos fricción tenga (voz, nota rápida en el móvil, correo a ti mismo). Después lo pasas al inbox.
- Si es del día: anótalo directamente en la nota diaria de [[Diario]].

**Ejemplo práctico**: En una reunión el cliente menciona que le gustaría recibir un presupuesto de neumáticos antes del verano. Capturas: "Cliente X — presupuesto neumáticos verano" en Captura rápida.

---

### 2. Aclarar

Cuando revisas el inbox, para cada elemento te preguntas: ¿qué es esto exactamente? ¿Qué acción implica, si es que implica alguna?

- ¿Es algo que tienes que hacer? → Es una tarea. Va al proyecto o a la nota diaria correspondiente.
- ¿Es algo que quieres aprender o investigar? → Es conocimiento. Va a Investigaciones o a una nota de área.
- ¿Es una idea sin desarrollar? → Va a Ideas.
- ¿Es un contacto relevante? → Va a Personas.
- ¿Es referencia pura? → Va a Recursos.
- ¿No vale nada ahora mismo? → Se archiva o se elimina.

---

### 3. Clasificar

Una vez aclarado, el elemento va a su lugar:

| Tipo | Destino |
|---|---|
| Proyecto nuevo o hito de proyecto | `Proyectos/` |
| Conocimiento o aprendizaje en profundidad | `Investigaciones/` |
| Idea embrionaria | `Ideas/` |
| Contacto relevante | `Personas/` |
| Responsabilidad continua | `Areas/` |
| Referencia o documentación | `Recursos/` |
| Reflexión del día | `Diario/` |

---

### 4. Conectar

Una nota aislada tiene poco valor. Conectar es donde el sistema empieza a trabajar para ti.

Al crear o actualizar una nota:
- Añade `[[wikilinks]]` a notas relacionadas
- Desde los MOCs ([[MOC - Inicio]], [[MOC - Trabajo]], [[MOC - PlayersHubs]], [[MOC - Aprendizaje]]), enlaza las notas relevantes
- En proyectos e investigaciones, enlaza ideas relacionadas y viceversa

**Ejemplo práctico**: Creas una investigación sobre segmentación de clientes. La enlazas desde [[Areas/Trabajo Postventa/Area - Trabajo Postventa|Trabajo Postventa]], desde [[MOC - Trabajo]] y desde la idea de [[Ideas/Idea - Automatizacion avanzada de postventa|Automatización de postventa]].

---

### 5. Ejecutar

El sistema sirve para actuar, no para acumular notas.

- La nota diaria de [[Diario]] define el foco y las tareas del día
- Cada proyecto tiene una "siguiente acción" clara y actualizada
- Las áreas tienen temas activos con acciones pendientes
- El MOC de inicio tiene una checklist de revisión diaria

---

### 6. Revisar

Sin revisión el sistema se pudre. La revisión es lo que mantiene todo vivo y útil.

- **Diaria** (5-10 min): Revisar el inbox, actualizar la nota del día, confirmar las prioridades
- **Semanal** (20-30 min): Checklist completa en [[Sistema/Revisiones semanales|Revisiones semanales]]
- **Mensual** (opcional): Revisar áreas y proyectos, cerrar lo que ya no esté activo, identificar patrones

---

## Lo que hace que este sistema funcione

1. **Baja fricción para capturar**: si capturar es difícil, no capturas. El inbox existe para eso.
2. **Un lugar para cada cosa**: no hay que pensar dónde va algo si las categorías son claras.
3. **Conexiones útiles**: un wikilink que realmente llevas a algo relacionado vale mucho más que un tag genérico.
4. **Revisión constante**: sin ella, el sistema pierde vigencia.
