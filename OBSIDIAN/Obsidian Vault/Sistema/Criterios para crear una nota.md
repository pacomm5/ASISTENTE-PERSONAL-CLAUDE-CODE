# Criterios para crear una nota

No todo merece una nota. Crear notas sin criterio infla el sistema y hace más difícil encontrar lo que importa.

---

## Cuándo SÍ merece la pena crear una nota

### Hay valor futuro claro

Si sabes que volverás a necesitar esta información, merece su propio espacio. Si no puedes imaginar un escenario en el que vuelvas a ella, no la crees.

Ejemplos que sí:
- Documentación de una herramienta que vas a usar en varios proyectos
- Análisis de un cliente con quien tienes relación continua
- Decisión importante con su razonamiento

### Hay posibilidad de reutilización

Si el contenido puede aplicarse en distintos contextos o servir de base para trabajo futuro, vale la pena estructurarlo.

Ejemplos que sí:
- Un script de Python que resuelve un problema que se repetirá
- Una plantilla de propuesta para clientes
- Un marco de análisis para segmentación

### Existe relación con otros temas

Si el contenido conecta con proyectos, ideas o investigaciones existentes, una nota separada facilita esas conexiones con wikilinks.

Ejemplos que sí:
- Una investigación sobre arquitectura de apps que se enlaza con PlayersHubs
- Una idea de campaña que se conecta con el área de trabajo

### Puede convertirse en acción, proyecto o referencia

Si el elemento tiene el potencial de crecer o de generar trabajo concreto, merece su propio espacio desde el principio.

---

## Cuándo NO merece la pena crear una nota

| Situación | Qué hacer en su lugar |
|---|---|
| Información efímera (fecha de una reunión puntual) | Nota diaria, sin nota propia |
| Dato que consultas una vez y no volverás a necesitar | No capturar o anotar en el diario |
| Información que ya está documentada en otro sitio | Enlazar a la fuente, no duplicar |
| Idea vaga sin ningún desarrollo posible | Desecharla o anotarla en el inbox para revisarla después |
| Variante menor de algo que ya existe | Actualizar la nota existente |

---

## La prueba del mes

Antes de crear una nota, hazte esta pregunta: ¿en un mes, si busco sobre este tema, quiero encontrar esta nota?

Si la respuesta es sí, créala. Si dudas, anótala en el inbox y decide en la próxima revisión.

---

## Señales de que una nota existente necesita dividirse

- Tiene más de 3 secciones principales sin relación directa entre ellas
- Trata temas que cada uno podría ser un proyecto o área distinto
- Te resulta difícil ponerle un título que lo describa todo
