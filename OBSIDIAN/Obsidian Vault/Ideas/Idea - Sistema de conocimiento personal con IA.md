# Sistema de conocimiento personal con IA

## La idea en una frase

Construir un segundo cerebro que combine Obsidian como repositorio permanente con Claude Code como asistente activo, de forma que el conocimiento acumulado sea cada vez más útil y el asistente cada vez más contextualizado.

---

## Problema que resuelve

El conocimiento que generamos trabajando, aprendiendo y tomando decisiones tiende a perderse o a quedarse fragmentado. Las conversaciones con IA terminan y el contexto desaparece. Las notas se acumulan sin conectarse. Las decisiones se toman sin memoria de las anteriores.

Este sistema resuelve:
- La pérdida de contexto entre conversaciones con el asistente
- La fragmentación del conocimiento entre herramientas
- La falta de conexión entre aprendizajes, decisiones y proyectos
- La incapacidad de reutilizar lo que ya se ha pensado o aprendido

---

## Por qué importa

Un sistema de conocimiento bien construido es un activo que se aprecia con el tiempo. Cada decisión documentada, cada aprendizaje capturado, cada conexión entre notas añade valor al sistema completo. Con IA integrada, el asistente puede orientarse más rápido, hacer mejores sugerencias y generar outputs más relevantes.

Para alguien que trabaja en múltiples frentes simultáneos (dirección de postventa, desarrollo de PlayersHubs, aprendizaje continuo), tener un sistema así reduce la carga cognitiva y permite actuar con más claridad.

---

## Contexto

Esta idea ya está siendo implementada: este vault es la respuesta concreta a esta idea. La investigación relacionada documenta lo aprendido en el proceso. La idea sigue abierta porque hay mucho por desarrollar.

---

## Posibles componentes del sistema

**Obsidian como capa de persistencia**
- Notas conectadas con wikilinks
- Frontmatter con metadatos estructurados
- MOCs como mapas de navegación
- Templates para reducir fricción al capturar

**Claude Code como capa activa**
- CLAUDE.md y archivos de contexto para que el asistente se oriente
- Skills especializadas para tareas recurrentes
- Capacidad de leer, crear y actualizar notas del vault
- Documentación de decisiones y aprendizajes durante el trabajo

**Flujo de trabajo**
- Captura → Inbox
- Procesado → Clasificar y conectar
- Revisión semanal → Mantener vigencia
- Cierre de proyectos → Archivar y extraer aprendizajes

**Evolución futura posible**
- Integración con Notion vía MCP para datos de proyectos
- Scripts de análisis del vault (notas más enlazadas, temas frecuentes)
- Automatización de la revisión semanal con un checklist activo asistido por IA
- Exportación de conocimiento a formatos reutilizables (presentaciones, informes)

---

## Riesgos o dudas

- El sistema puede crecer tanto que se vuelva difícil de mantener (más fricción que valor)
- La tentación de perfeccionar el sistema en lugar de usarlo para producir resultados reales
- Dependencia excesiva del asistente para pensar, en lugar de usarlo para ejecutar
- Sin revisión constante, la bóveda envejece y pierde precisión
