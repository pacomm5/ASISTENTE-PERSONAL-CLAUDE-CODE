# Area — Programación

## Propósito

Desarrollo de software, automatizaciones y flujos técnicos. Incluye tanto proyectos personales (PlayersHubs) como herramientas para el trabajo (scripts, análisis, automatizaciones de postventa).

---

## Responsabilidades

- Desarrollo y mantenimiento de PlayersHubs
- Scripts de análisis y automatización para Jarmauto
- Exploración y aprendizaje de nuevas herramientas y arquitecturas
- Uso eficiente de Claude Code como asistente de desarrollo
- Documentación técnica de decisiones y arquitecturas

---

## Temas activos

- Definición de arquitectura inicial de PlayersHubs
- Configuración del entorno Docker para PlayersHubs
- Exploración de patrones de API REST para el backend

---

## Stack técnico

| Herramienta | Uso |
|---|---|
| Python | Scripts de análisis, automatizaciones, backend (FastAPI) |
| JavaScript / TypeScript | Frontend (React / Next.js) |
| Docker | Entornos reproducibles y despliegue |
| PostgreSQL | Base de datos relacional |
| Railway | Despliegue de proyectos propios |
| Google Cloud | Cloud para proyectos avanzados |
| Claude Code | Asistente de desarrollo y documentación |
| VS Code | Editor principal |
| GitHub | Control de versiones |

---

## Métricas o señales

- Avance semanal en PlayersHubs (hay commits o no)
- Número de scripts activos en producción para Jarmauto
- Tiempo dedicado a aprendizaje técnico vs producción
